# Claire Assistant — 会议/活动筹备模板管理

## 项目概述

为老板管理 Word 版会议/活动筹备模板。模板固定（5 大板块），用户通过**自然语言碎片化更新**，Claude 自动维护 JSON 结构化数据，按需生成 Word 文档。

## 核心文件

| 文件 | 作用 |
|---|---|
| `template.docx` | Word 模板（含 `{{占位符}}`，可自由修改布局） |
| `data/events.json` | 所有活动的结构化数据（唯一真相源） |
| `generate_docx.py` | 按名称匹配 `{{占位符}}` 生成 .docx |
| `output/` | 生成的 Word 文档输出目录 |

## 交互协议

### 新增活动
用户可以说：
- 「新建活动：7月15日微软AI峰会」
- 「创建一个活动，叫"清华学术交流"」

操作：在 `events.json` 的 `events` 数组中追加新活动记录，id 用拼音/英文 slug。

### 更新字段（自然语言匹配）
用户告知碎片信息，Claude 自动识别更新到对应字段：

- 「微软AI峰会：会议地点=北京国家会议中心」→ 更新 section1 的会议地点
- 「微软AI峰会：规模大约500人」→ 更新会议规模
- 「微软AI峰会背景：这是微软与政府的年度合作交流活动」→ 更新会议背景
- 「7/15活动大交通：去程CA1234北京-上海，回程CA1235上海-北京」→ 更新去程/回程
- 「微软AI峰会的酒店：国贸大酒店，确认号CN12345」→ 更新酒店安排子项
- 「微软AI峰会衣着：商务正装」→ 更新衣着要求
- 「微软AI峰会：在现场参加前面打勾」→ 更新 会议属性.宣传安排.现场参加 = true
- 「微软AI峰会：勾选微软内部、现场参加」→ 可同时勾选多个选项

### 添加日程
- 「微软AI峰会加日程：7月15日 9:00-10:00 开幕式主题演讲」
- 「微软AI峰会 7/15日程增加：14:00-15:30 圆桌讨论」

多天日程的每一天是独立的 日程 对象，格式：
```json
{ "日期": "7月15日", "条目": [{ "时间": "9:00-10:00", "内容": "开幕式" }] }
```

### 添加要员
- 「微软AI峰会加要员：张三，微软亚研院院长，近期做了GPT-5研究」
- 「微软AI峰会加要员：李四，教育部副部长」

格式：
```json
{ "姓名": "张三", "Title公司": "微软亚洲研究院院长", "简介近期动态": "近期发布了GPT相关论文" }
```

### 添加待办
- 「微软AI峰会待办：确认韦总是否有特殊饮食要求」
- 「微软AI峰会待办：准备中英文双语PPT」

### 移除/删除
- 「删除微软AI峰会日程里的圆桌讨论」
- 「删除微软AI峰会要员张三」

### 生成 Word
- 「生成微软AI峰会的Word」
- 「生成所有活动的文档」

### 要员简介查询
- 「刷新 [姓名] 简介」（如「刷新周鸿祎简介」）→ 自动在所有活动中搜索该姓名的要员，刷新所有匹配项
- 「刷新要员简介 [event-id] [姓名] [职务/公司]」→ 精确指定活动ID的单个要员
- 「刷新 XX 活动所有要员简介」→ 批量查询该活动所有要员
- 「刷新所有要员简介」→ 批量查询所有活动所有要员

要员简介查询规则：
1. **触发条件**：要员必须有「姓名」和「Title公司」两项都有值时才可查询
2. **数据来源**：通用网络搜索（Bing、百度等）
3. **输出格式**：分「个人简介」「近期动态」两部分，每部分约 300 字，使用项目符号
4. **信息不足时**：写「网上信息源不足」
5. **数据写入**：`section2_schedule.要员简介[].简介近期动态` 字段
6. **查询完成后**：自动 commit + push 到 GitHub，用户在表单页按 F5 刷新即可看到结果
7. **表单按钮**：活动表单「要员简介」单元格里每个要员旁有「🔍 查询」按钮，点击复制触发短语；用户也可直接在对话中说「刷新 [姓名] 简介」

### 天气查询
- 「刷新天气」→ 刷新所有活动的目的地天气
- 「刷新 XX 活动的天气」→ 刷新指定活动的目的地天气

天气查询规则：
1. **触发条件**：活动必须有 会议地点、会议时间、会议名称 三项都有值时才可查询
2. **数据来源**：weather.com.cn（中国天气网），仅支持中国大陆城市
3. **时间范围**：不限。任何时候都可查询，查询当时可获得的预报信息即写入；若 weather.com.cn 对该日期无预报（例如活动日期距当前超过7天，超出中国天气网的预报范围），写入「暂无预报数据（更新于X月X日）」
4. **生成文档时自动检查**：如果目的地天气为空且条件满足，自动查询后写入
5. **输出格式**：`[城市][月日]（[周几]）[天气情况]，[最低温]-[最高温]℃，[风向]风[风力]级，降水概率约[XX]%。（更新于[月日]）`
6. 天气数据写入 `section4_departure.目的地天气` 字段
7. **临近活动可再次刷新**：因为早期查询到的可能是「暂无预报数据」，活动临近（7天内）后用户可再次点刷新获取真实预报

### 出行提示查询
- 与天气查询联动触发，条件相同：会议地点、会议时间、会议名称 三项都有值
- 「刷新交通」→ 刷新所有活动的出行提示
- 「刷新 XX 活动的交通」→ 刷新指定活动的出行提示

出行提示查询规则：
1. **触发条件**：与天气查询相同，三项必备
2. **数据来源**：优先从城市交管局官网查询，其次从百度地图查询
3. **时间范围**：不限。任何时候都可查询；若查不到该日期的限行/管制信息，写入「暂无交通信息（更新于X月X日）」
4. **生成文档时自动检查**：出行提示为空且条件满足时，自动查询
5. **输出格式**：`[城市]城区[月日]限行尾号[XX]，[有无大型活动]，[有无交通管制信息]。（更新于[月日]）`
6. 交通数据写入 `section4_departure.出行提示` 字段
7. 若某项信息查不到，该单项显示「暂无」
8. **临近活动可再次刷新**：早期可能查不到限行尾号（交管局通常只提前7天公布），活动临近后再点刷新即可

### 本地直接触发（页面按钮 → Claude）
当本机运行着 `local_refresh_server.py` 时，GitHub Pages 表单页面顶部的 🌤/🚗/👥 按钮可以**直接**触发本机 Claude 执行刷新，无需切换到对话窗口粘贴短语。

**启动服务：**
- 双击 `start-refresh-server.bat`，会弹出一个控制台窗口，显示「Claire 本地刷新服务 监听 http://127.0.0.1:8787」
- 关闭控制台窗口即停止服务

**触发流程：**
1. 用户在 https://claireli2621.github.io/claire_event_calendar/event-form-demo.html?view=活动ID 打开活动详情页
2. 顶部会显示「🌤 天气 / 🚗 交通 / 👥 简介」三个按钮和连接状态
3. 点击按钮 → 页面 POST 到 `http://127.0.0.1:8787/refresh`，body 为 `{type, event_id, event_name}`
4. 本地服务调 `claude.exe -p "刷新 XX 活动的天气"` 等触发短语
5. Claude（本机）执行 SKILL → 写入 events.json → 推送 GitHub
6. 服务返回成功 → 页面 refetch events.json 并自动重填表单

**局限：**
- 只在运行服务的那台电脑上有效；其他人访问页面时按钮会显示「本机未启动刷新服务」
- Claude 必须已登录（用 NiuClaude 启动过即可，auth 文件已就绪）
- HTTPS 页面调 http://localhost 是浏览器允许的安全例外，不会出现混合内容拦截

## JSON 数据结构说明

每个活动 5 个 section，与模板一一对应：
1. `section1_basic_info` — 活动基本信息（含 会议属性 子对象）
2. `section2_schedule` — 活动日程（日程数组 + 要员简介数组）
3. `section3_preparation` — 会议准备内容（含 PPT要求 子对象、待办数组）
4. `section4_departure` — 出发准备（含 酒店安排 子对象）
5. `section5_wrapup` — 会后收尾

未填字段保留空字符串 `""`，生成 Word 时会显示 `xxxx` 占位符。

## 更新规则

1. **每次更新后立即保存 JSON**（不延迟写入）
2. **匹配不精确时主动确认**（如用户说的活动名称匹配多个，则列出候选）
3. **字段值覆盖式更新**（新信息替换旧值；如需拼接，从上下文判断）
4. **更新 `updated` 时间戳**
5. **生成 Word 时自动补齐所有 5 个 section**（包括空字段显示占位符）
